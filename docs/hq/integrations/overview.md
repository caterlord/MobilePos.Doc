:::info[Who this is for]
Admins who connect X1 HQ to external partners and maintain those partner connections.
:::

## Before you start

- Confirm the correct brand.
- Confirm whether the partner is already subscribed or installed through `Marketplace`.
- Gather any identifiers, tokens, or partner-provided setup details before editing.

## What this section is for

Use `Integrations` to manage partner connections between X1 and third-party services.

This section is different from shop hardware setup:

- use `Integrations` for partner connections and partner shop configuration
- use `Store Settings` -> `Device Settings` for local device and terminal setup

## How to use this section

- [Partner Settings](./partner-settings.md): review installed partners, add new ones, and manage partner setup per shop.

If you also need to enable optional services before configuration, use [Marketplace](../marketplace/catalog.md).

## Common tasks

- Connect a new partner after subscription: start with [Partner Settings](./partner-settings.md).
- Check whether a partner is configured for the right shop: use [Partner Settings](./partner-settings.md).
- Confirm whether the partner needs marketplace subscription first: review [Marketplace](../marketplace/catalog.md).

## What changes after you save

Integration setup can depend on more than one step:

- marketplace subscription
- partner installation
- per-shop configuration

Save and verify each layer separately.

## If something goes wrong

- Confirm the partner is installed or subscribed first.
- Confirm the correct brand and shop.
- Check whether the issue is in partner setup or in a different section such as `Payment Vendors` or `Device Settings`.

## When to ask owner/admin

- You do not have the partner credentials.
- The integration affects finance, online ordering, or live checkout.
- The partner is installed but no data is moving as expected.

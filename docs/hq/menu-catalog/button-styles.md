title: Button Styles
---

:::info[Who this is for]
Admins who manage reusable button appearance presets for category or item display.
:::

## Before you start

- Confirm the correct brand.
- Decide whether you need a new style or can reuse an existing one.

## Open this page

Open `Menu Management` -> `Button Styles`.

## What this page is for

`Button Styles` stores reusable visual styles that can be applied to buttons elsewhere in the menu setup.

Use this page to:

- add a new button style
- review existing styles
- update a non-system style
- remove a non-system style you no longer need
- create a missing style directly from a `Button Style` selector on supported menu setup pages

## Main sections

### Styles table

The main table includes:

- `ID`
- `Style Name`
- `Preview`
- `Color Code`

System styles are marked with a `System` badge and cannot be edited or deleted.

### Style editor

The editor includes fields such as:

- `Style Name`
- `Background Color`

Key actions include:

- `Add Button Style`
- `Create`
- `Update`
- `Delete`

## Steps

1. Open `Menu Management` -> `Button Styles`.
2. Select `Add Button Style` to create a new style, or open an existing style to edit.
3. Enter `Style Name`.
4. Select the `Background Color`.
5. Check the preview.
6. Save the style.

## Create a style while editing menu setup

Supported `Button Style` selectors also include a create-style action.

1. Open a category, smart category, online category, or menu item editor.
2. Find `Button Style`.
3. Select `Create button style`.
4. Enter `Style Name`.
5. Select `Color Palette`.
6. Check `Preview`.
7. Select `Create Style`.
8. Confirm the new style is selected in `Button Style`.

## What changes after you save

Saving the style updates the reusable preset. It does not automatically prove that every related category or item is using it correctly, so verify where the style is applied.

## How to check your change

1. Return to the styles table and confirm the style is listed.
2. Check the preview and color.
3. Open the category or other setup that uses the style and confirm the correct style is selected.

## If something goes wrong

- Confirm you are not trying to edit a `System` style.
- Check whether the style was saved but not yet assigned to a category or item.
- If the style is no longer needed, remove it only after confirming it is not still in use.

## When to ask owner/admin

- The style change would affect multiple live menus and you cannot confirm all of them.
- You are not sure whether a shared style is used by other shops or teams.

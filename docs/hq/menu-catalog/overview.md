:::info Who this is for
Admins who create and maintain sellable items, menu structure, pricing logic, and cashier-facing catalog behavior.
:::

## Before you start

- Confirm the correct brand.
- Decide whether you are changing structure, item content, pricing logic, or POS display behavior.
- If the change affects live service, record the current state before editing.

## What this section is for

`Menu Management` is the core catalog setup area for HQ.

Use this section to manage:

- categories and smart categories
- menu items
- POS menus
- modifiers
- meal sets
- promotions and discounts
- button styles used in the interface

## How to use this section

Use the pages in this section in a logical setup order:

1. [Button Styles](./button-styles.md): create reusable button appearance presets when needed.
2. [Categories](./categories.md): define the standard category structure.
3. [Smart Categories](./smart-categories.md): create rule-based or display-specific category groupings.
4. [Menu Items](./menu-items.md): create the products the POS can sell.
5. [Modifiers](./modifiers.md): define add-on or selection groups.
6. [Meal Sets](./meal-sets.md): create combo-style selection groups.
7. [POS Menus](./pos-menus.md): decide which categories appear together in a menu.
8. [Promotions](./promotions.md) and [Discounts](./discounts.md): define automatic or manual price reductions.

## Common tasks

- Create a new sellable item: start with [Menu Items](./menu-items.md).
- Change how cashiers browse products: review [Categories](./categories.md), [Smart Categories](./smart-categories.md), and [POS Menus](./pos-menus.md).
- Add combo choices: use [Meal Sets](./meal-sets.md) with [Menu Items](./menu-items.md).
- Add manual markdown rules: use [Discounts](./discounts.md).
- Add automatic cart logic: use [Promotions](./promotions.md).

## What changes after you save

Catalog changes can affect cashier browsing, reporting structure, and online ordering setup. Save one related change at a time and verify the result in the target workflow.

## If something goes wrong

- Confirm the item or category is attached to the correct parent structure.
- Check whether the issue is in categories, menus, or item settings before editing more records.
- Verify whether a customer-facing channel uses a separate online ordering configuration after the core menu change.

## When to ask owner/admin

- You are changing live pricing, discounting, or promotional rules.
- The same item structure affects multiple shops and channels.
- You cannot tell whether the issue is in the base catalog or in online ordering setup.

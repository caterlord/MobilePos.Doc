---
title: Change Table
---

:::info[Who this is for]
Cashiers and shift managers who need to move an open order to another table.
:::

## Before you start

- You are signed in and on the order screen.
- You have opened the correct order.
- You can open the function menu (three dots).
- You have sent pending item edits with `Send`.
- You know the destination is an empty table.

## Steps

1. Tap function menu (three dots).
2. In `Select Function`, tap `Change Table`.
3. If `Change Table` is on another page, tap `Previous` or `Next`.
4. In `Change Table`, tap the lower-right toggle if you need to switch between `Table List` and `Floor Plan`.
5. If you need split tables, tap `Show Split`.
6. Tap the destination table.
7. Wait for the table selector to close.
8. Open the moved order from the destination table if the app returns to the table list.

## What you should see

- The order is no longer on the original table.
- The order appears on the destination table.
- You can continue ordering on the destination table.
- The lower-right toggle switches between `Table List` and `Floor Plan` when floor plan mode is enabled.
- `Show Split` changes to `Hide Split` after split tables are shown.

## If something goes wrong

- If you see `No item to change table`, add at least one item first.
- If you see `Cannot change table` with `Please save all operations before changing table`, tap `Send`, then retry.
- If you cannot find the target table in `Floor Plan`, switch to `Table List`.
- If the target split table is missing, tap `Show Split`.
- If you see `Please select an empty table`, choose another table.
- If you see `Unable to change table`, retry once after refreshing the order.

## When to ask manager/admin

- `Manager only`: move orders tied to special handling or dispute cases. Cashier fallback: keep the current table and continue service.
- Ask a manager if no valid destination table is available.
- Ask admin if `Change Table` keeps failing after one retry.

---
title: Split or Merge Transaction
---

:::info[Who this is for]
Cashiers and shift managers who need to move items between transactions before payment.
:::

## Before you start

- You are signed in and on the order screen.
- You can open the function menu (three dots).
- You have sent pending item edits with `Send`.
- You understand the difference:
`Split / Merge Txn` is for moving items between transactions before checkout.
`Split By Item` is on checkout and is for payment split flow.

## Steps

1. Tap function menu (three dots).
2. In `Select Function`, tap `Split / Merge Txn`.
3. If `Split / Merge Txn` is on another page, tap `Previous` or `Next`.
4. Tap the target table in `Select Table`.
5. Tap an item on the source side to select it.
6. Tap `>` to move one selected item, or tap `All` to move all items.
7. Tap `Confirm`.
8. Open the target transaction and verify moved items.

## What you should see

- The `Split / Merge Txn` dialog opens with source and target panels.
- Moved items appear on the target side before you confirm.
- After `Confirm`, the new transaction state is saved.

## If something goes wrong

- If you see `Have unsend item(s), cannot split Txn`, tap `Send`, then retry.
- If you see `Please transfer the item`, move at least one item before `Confirm`.
- If you see `Table is placing order`, pick another target table.
- If you see `Unable to split order`, retry once.

## When to ask manager/admin

- `Manager only`: merge/split across tables when store policy requires approval. Cashier fallback: keep current transaction and continue service.
- Ask a manager if you are unsure whether to use `Split / Merge Txn` or checkout `Split By Item`.
- Ask admin if split/merge fails repeatedly on valid tables.

---
title: To Dine-In or To Takeaway
---

## Who this is for

Cashiers and shift managers who need to switch an order type between dine-in and takeaway.

## Before you start

- You are signed in and on the order screen.
- You can open the function menu (three dots).
- You have opened the correct order.
- You know the required order type for this customer.

## Steps

1. Tap function menu (three dots).
2. Tap `To Dine-In` or `To Takeaway` based on the current order type.
3. Check that the order type changes on the screen.
4. Tap `Send` to save the updated order.

## What you should see

- The function label flips after switching:
if you switched to dine-in, you will next see `To Takeaway`;
if you switched to takeaway, you will next see `To Dine-In`.
- The order stays open so you can continue service.
- Screenshot pending: `fn-switch-order-type-function-menu.png`
- Screenshot pending: `fn-switch-order-type-result.png`

## If something goes wrong

- If `To Dine-In` or `To Takeaway` is missing, retry on the same order after refresh.
- If order totals change unexpectedly after switching, stop and verify before checkout.
- If the order type does not stay updated, tap `Send` and retry once.

## When to ask manager/admin

- `Manager only`: change order type after kitchen workflow has already started, if your store policy requires approval. Cashier fallback: keep current type and continue service.
- Ask a manager when the requested order type conflicts with store rules.
- Ask admin when the switch option is missing for all users.

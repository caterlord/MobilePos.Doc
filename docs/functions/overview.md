:::info Why this section exists
Use-case pages are best for training. Function pages are best when you need one exact operation quickly.
:::

Use this section when you want one task per page, such as:

- Ordering functions (discount, service charge, change table, order-type switch, split/merge, member login)
- Checkout and completed-order functions (take payment, void/return, reopen, reprint)
- Devices functions (printer management, payments/input/X1 configuration)
- More Tools functions (pay in/out, day-end)

For full coverage index, use [Function and Feature Map](../reference/function-feature-map).

:::info Who this is for
Cashiers and shift managers running daily store operations.
:::

## Before you start

- Terminal is online
- Staff accounts are available
- Printer is ready

## SOP checklists

<Tabs>
  <TabItem value="opening" label="Opening">
1. Sign in and `Clock In` (if enabled).
2. Open `Devices` and run `Test Print`.
3. Confirm shop name and business date/time.
4. Open `Ordering` and verify table screen loads.
5. Open checkout once and verify payment methods are available.
  </TabItem>
  <TabItem value="during-shift" label="During shift">
1. Send orders promptly after item confirmation.
2. Re-check device status after printer/network alerts.
3. Use split payment modes only when customer requests.
4. Add remarks when policy requires audit trail.
5. Assign one owner to resolve each payment incident.
  </TabItem>
  <TabItem value="handover" label="Handover">
1. Stop creating new orders on handover terminal.
2. Complete or clearly hand over in-progress checkouts.
3. Confirm pending void/return items are manager-approved.
4. Hand over unresolved incidents with transaction IDs.
  </TabItem>
  <TabItem value="closing" label="Closing">
1. Resolve active orders and pending payments.
2. Confirm no cashier is stuck in checkout processing.
3. Complete pay in/out entries.
4. Ensure manager is ready for day-end.
5. Run day-end and confirm completion.
  </TabItem>
</Tabs>

## What good operations look like

- No blocked printer/payment issues at shift start
- Clear ownership for any incident
- No unresolved checkout at day-end

## If something goes wrong

:::warning Payment status is unclear
Stop repeated retries and escalate immediately to manager.
:::

:::warning Device remains unavailable
Run basic checks once, then escalate instead of repeated attempts.
:::

## When to ask manager/admin

- `POS Code` or critical settings must be changed
- Refund/void requires approval
- Day-end warning/validation blocks closure

This manual is for the **POS X1 Client app** used by store teams.

This English draft is written from the current `MobilePos` client source and UI labels.
Focus is cashier and manager operations in the Client app. HQ pages remain work in progress.

## Scope

- Daily sign-in and module navigation
- Table/order operations
- Checkout and payment handling
- Void/return/reprint workflows
- Manager tools and day-end
- Device health checks

## Source of Truth

- Client source: `https://github.com/caterlord/MobilePos`
- HQ source: `https://github.com/caterlord/MobilePos.HQ`

## How to Read

1. New store setup: **Getting Started**
2. Daily operation: **Client (POS App)**
3. Issue handling: **Reference**

## How the Content Is Organized

This manual now has three parallel views:

- **Use-case view**: step-by-step operation guides (start shift, sales, checkout, refunds, day-end)
- **Role view**: cashier vs manager permissions and escalation boundaries
- **Function view**: one function per page for quick lookup, plus a full feature index

Use **Function Guides** for one-task pages, and use **Reference -> Function & Feature Map** for full inventory.

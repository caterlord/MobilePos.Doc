# X1 MCP 可以做什么

:::info[适合对象]
想了解 AI 助理可通过 X1 MCP 协助哪些 HQ 工作的店主、经理和管理员。
:::

## 开始之前

- 将 AI 助理连接到 X1 MCP。
- 确认助理能列出正确品牌和门店。
- 在要求更改前，先使用只读问题。

## 核心概念

X1 MCP 将 HQ 工作变成受引导的助理流程：

1. 检查当前设置。
2. 准备建议更改。
3. 显示更改前后预览。
4. 等待批准。
5. 应用已批准更改。
6. 验证结果。

## 报表和分析

不用逐个打开 HQ 报表，也可以要求助理检查表现。

示例：

- “按门店显示最近 7 日销售。”
- “比较本周一和上周一。”
- “本周哪些分类增长最多？”
- “显示热饮品项表现。”
- “找出需要注意的运营异常。”

助理可以整理销售、趋势、期间比较、品项表现、分类表现和运营异常。

## 菜单创建和导入

当你有表格、PDF、图片或文字菜单来源时使用。

建议请求：

> 读取这份菜单，先显示分类、品项、价格、加配项和套餐规则，不要立即创建任何 X1 数据。

助理可以协助：

- 读取现有 X1 菜单结构
- 准备结构化菜单草稿
- 验证分类、品项、加配项和套餐规则
- 预览菜单导入
- 应用已批准导入

请仔细检查预览。上传菜单常有缩写、手写备注或价格规则，需要人工确认。

## 菜单维护

用于更改现有目录。

示例：

- “将所有热饮加价 `$1`。”
- “今天将这些品项设为在线点餐不可售。”
- “将这些品项从 `Lunch` 移到 `Dinner`。”
- “比较两家门店的菜单设置。”
- “显示差异后，将一家门店的菜单设置复制到另一家门店。”

助理可以读取品项、分类、加配项、智能分类和 POS 菜单。它可以预览和应用品项更改、价格调整、供应状态、重新分类和指定批量菜单更新。

## 加配项和套餐规则

当品项需要选项、加购或套餐规则时使用。

示例：

- “为咖啡创建牛奶选项加配组。”
- “创建套餐规则：任何 `Lunch` 品项包含一杯热饮，换冰饮加 `$3`。”
- “更改前显示哪些品项正在使用这个加配组。”

助理可以在你批准后预览和应用加配组更改和套餐规则更改。

## 在线点餐准备状态

发布或排查顾客可见菜单前使用。

示例：

- “检查这家门店是否已准备好在线点餐。”
- “为什么这个品项没有在线显示？”
- “显示缺少的分类、菜单或渠道设置。”

助理可以检查在线点餐设置、诊断菜单显示、检查发布准备状态，并预览指定在线点餐设置更改。

## 门店和设备设置

用于检查和小范围受控更改。

示例：

- “显示这家门店的门店设置快照。”
- “显示设备设置和终端配置。”
- “先显示当前名称，再重命名这个钱箱。”

助理可以检查门店设置、设备设置，并预览指定设备设置更改，例如钱箱名称。

## 后台任务状态

部分已批准更改可能以后台任务执行。

你可以问：

> 检查任务状态，并告诉我更改是否已成功完成。

在 HQ、POS 或在线点餐验证结果前，先完成这一步。

## 何时要求店主/管理员协助

- 助理建议跨多家门店更改。
- 更改影响实时价格、菜单显示、在线点餐、付款相关设置或设备。
- 预览没有清楚显示所有受影响记录。
- 来源菜单不清楚，或与当前 HQ 设置有冲突。

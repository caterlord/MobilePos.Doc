:::info[适用对象]
负责日常门店运营的收银员和班次经理。
:::

## 开始前

- 终端已联网
- 员工账号可用
- 打印机已就绪

## SOP 检查清单

<Tabs>
  <TabItem value="opening" label="开店">
1. 登录，并在启用时点击 `上班打卡`。
2. 打开 `设备`，执行一次 `测试打印`。
3. 确认门店名称与营业日期/时间正确。
4. 打开 `点餐`，确认桌台画面能正常载入。
5. 打开一次 `结账`，确认付款方式可用。
  </TabItem>
  <TabItem value="during-shift" label="营业中">
1. 确认项目后，尽快发送订单。
2. 收到打印机或网络警告后，重新检查设备状态。
3. 只有在顾客要求时，才使用分单付款。
4. 当制度要求留痕时，添加备注。
5. 每一笔付款异常都要指定一位负责人处理。
  </TabItem>
  <TabItem value="handover" label="交接">
1. 在交接终端上停止创建新订单。
2. 完成进行中的结账，或清楚交接给下一位同事。
3. 确认待作废/退回项目已获得经理批准。
4. 交接未解决问题时，附上交易编号。
  </TabItem>
  <TabItem value="closing" label="收店">
1. 先处理未完成订单和待付款项。
2. 确认没有收银员卡在结账处理中。
3. 完成所有 `代收 / 代支` 记录。
4. 确认经理已经准备好执行 `日结`。
5. 运行 `日结`，并确认完成。
  </TabItem>
</Tabs>

## 正常运营应该是什么样

- 开班时没有被打印机或付款问题阻挡
- 每一笔异常都有明确负责人
- 到日结时没有未解决的结账

## 出现问题时

:::warning[付款状态不明确]
不要重复重试，立刻上报经理。
:::

:::warning[设备仍然不可用]
先做一次基础检查，然后直接上报，不要反复尝试。
:::

## 何时找经理/管理员

- 需要修改 `收银机代码` 或关键设置
- 退款/作废需要审批
- `日结` 警告或验证阻止关闭

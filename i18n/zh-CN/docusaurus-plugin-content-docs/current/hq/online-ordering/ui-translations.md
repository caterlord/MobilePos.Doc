:::info[适用对象]
需要在不等待前台发布的情况下覆盖顾客端线上点餐文案的管理员。
:::

## 开始前

- 确认已选择正确的品牌。
- 先决定要修改哪个语言以及哪个前台渠道。
- 确认您要做的是界面文案覆盖，而不是基础菜单或项目名称翻译。

## 打开此页面

打开 `Online Ordering` -> `UI i18n`。

## 本页用途

`UI i18n` 可让您覆盖线上点餐应用中的顾客前台文案。

您可以在这里：

- 选择要修改的语言
- 指定某个订单渠道
- 搜索要修改的文案 key
- 保存自定义覆盖文本
- 删除覆盖并恢复系统默认文案

本页只会修改前台界面文本，例如按钮、标签和提示，不会替代其他位置维护的菜单项目翻译。

![UI 翻译](/img/hq/online-ordering/oo_translations_page.png)

## 主要区域

### 语言选择

选择要修改的语言，例如 `Traditional Chinese (HK)`。

### 订单渠道选择

选择要应用覆盖的前台或订单渠道，例如 `WEB`。

### Key 搜索

使用搜索栏找出要修改的文案 key。

如果您不知道 key 名称，可以先搜索顾客实际看到的词语，例如 `Dine-In`。

### 覆盖值

在覆盖字段中输入新的文本，然后保存。

如果您想删除自定义覆盖并恢复系统默认文案，请先把覆盖字段完全清空，再点击保存。

## 步骤

1. 打开 `Online Ordering` -> `UI i18n`。
2. 选择目标语言。
3. 选择目标订单渠道。
4. 搜索要修改的文字 key。
5. 输入新的覆盖值。
6. 选择 `Save`。

## 保存后的影响

目标前台会在指定语言与订单渠道中使用新的文本。

这类变更只会更新顾客前台界面文案，不需要重新部署前台程序。

## 如何检查你的变更

1. 重新打开相同 key，确认覆盖值仍然存在。
2. 以相同语言与渠道打开目标前台。
3. 确认页面对应位置已显示新文案。

## 如发生问题

- 确认您修改的是正确语言与订单渠道。
- 确认您修改的是正确 key，而不是另一个相似标签。
- 如果您想删除覆盖，请先把字段完全清空再保存。
- 如果前台仍显示旧文本，请刷新目标流程并再次确认渠道选择。

## 何时请店主／管理员协助

- 文案变更会影响营业中的顾客流程。
- 您不确定这应该是本地覆盖，还是永久产品默认值。
- 文案修改涉及法规、支付或合规风险。

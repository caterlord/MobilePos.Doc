:::info[适用对象]
负责建立或维护 POS 内销售、客制化或追踪商品的管理员。
:::

## 开始前

- 确认正确的品牌。
- 先决定这笔纪录是可售商品、加购商品，还是组合商品。
- 保存前，请先确认目标分类和部门。
- 如果要复制商品，请先准备未被启用商品使用的全新 `Item Code`。

## 打开此页面

打开 `菜单管理` -> `菜单项目`。

## 本页控制内容

`菜单项目` 是 POS 菜单的主要商品资料库。

您可以在这里建立或维护：

- 可售商品
- 加购商品
- 套餐或组合父商品
- 商品定价和可售状态
- 特殊商品行为
- 基于现有商品设置建立的新商品副本

![菜单项目列表](/img/hq/menu-catalog/menu_items_list.png)

编辑商品时，抽屉标题会显示当前的 `Item Code` 和 `Item Name`。如果使用 `Skip to Next` 或 `Update & Next`，抽屉会保留当前选取的分页，方便你在多笔记录之间持续处理同一类设置。

## 主要分区

### 商品类型

继续之前，请先选对商品类型：

- 可售商品
- 加购项
- 套餐商品

商品类型会影响商品属于哪里，以及 POS 如何使用它。

### 基本栏位

| Field | What it controls | 何时更改 |
| --- | --- | --- |
| `Item Code` | 商品内部识别码 | SKU 或整合代码变更时 |
| `Item Name` | HQ 的主要名称 | 核心纪录名称变更时 |
| `POS Display Name` | 员工在 POS 上看到的名称 | 销售名称与 HQ 名称不同时 |
| `Printed Name` | 收据或厨房输出上打印的名称 | 您需要较短或更适合运营的文字时 |
| 分类和部门栏位 | 浏览结构和报表结构 | 商品应该归到不同菜单位置或汇总层级时 |

如果建立或编辑商品时所需 `按钮样式` 尚未存在，请使用选择器旁的 `创建按钮样式`，输入 `样式名称`，选择 `调色板`，检查 `预览`，然后选择 `创建样式`。

### 特殊行为开关

请谨慎使用商品开关，因为它们会改变 POS 在结帐时的行为。

例如：

- 手动价格
- 不可折扣
- 不收服务费
- 不课税
- 收据显示选项

### 定价、可用性和门店设置

使用商品的定价、可用性和门店设置区块，来控制各分店销售价格、各门店是否启用，以及商品是否缺货。

### 复制现有商品

当新商品应从现有商品设置开始建立时，请使用复制动作。

1. 找到要复制的商品。
2. 在 `Actions` 栏选择复制动作。
3. 输入新的 `Item Code`、`Item Name`、`Item Name (Alt)`、`Item Kitchen Name` 和 `Item Kitchen Name (Alt)`。
4. 确认目标 `Category` 和 `Department`。
5. 保存副本。

复制会带入来源商品的相关设置，例如门店资料、价格、加购组、套餐群组和其他商品关系。

### 猛兽模式复制

如果需要从同一个来源商品一次建立多个新商品，请在复制窗口打开 `Beast Mode`。

打开 `Beast Mode` 后，每一行代表一个目标商品：

- `Item Code`
- `Item Name`
- `Item Name (Alt)`
- `Item Kitchen Name`
- `Item Kitchen Name (Alt)`

同一次保存中的所有行会共用同一个目标 `Category` 和 `Department`。

保存前，请确认每个 `Item Code` 在行列表中不重复，且没有被任何启用中的商品使用。只要其中一个目标商品无法保存，本次所有目标商品都不会建立。

## 步骤

1. 打开 `菜单管理` -> `菜单项目`。
2. 新增商品，或打开现有商品。
3. 选择正确的商品类型。
4. 输入商品识别栏位。
5. 将商品指派到正确的分类和部门。
6. 套用需要的特殊行为开关。
7. 检查目标门店的定价和可用性。
8. 保存商品。

## 保存后的影响

这些更改会影响共用商品纪录，也可能影响门店可用性、POS 浏览、报表和结帐行为，视您更改的内容而定。

## 如何检查你的更改

1. 重新打开商品，确认保存后的值正确。
2. 确认商品位于正确的分类和部门。
3. 如果您更改了目标门店的价格、可用性或门店设置，也请检查这些设置。
4. 如果您更改了特殊开关，请测试受影响的 POS 行为。
5. 如果您复制了商品，请重新打开复制后的商品，确认相关设置已按预期复制。

## 如果出现问题

- 先确认错误结果不是由分类、部门或可用性设置造成。
- 如果 POS 行为不如预期，先检查商品类型。
- 如果商品存在，但在某一间门店缺失，请检查该门店的价格或可用性设置。
- 如果复制失败，请检查复制列表中或启用中的商品是否已有相同的 `Item Code`。

## 何时需要找老板／管理员

- 更改会影响现场定价或顾客可见的收据行为。
- 您不确定该商品应该是可售商品、加购项，还是套餐商品。

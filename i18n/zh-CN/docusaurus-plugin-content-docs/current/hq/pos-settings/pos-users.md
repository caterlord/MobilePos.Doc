:::info[适用对象]
负责维护 POS 员工登录纪录的管理员和经理。
:::

## 开始前

- 确认正确的品牌和目标门店。
- 先决定您是要建立用户群组，还是单一 POS 用户。
- 决定该群组可在 POS 执行哪些操作。
- 不要把 POS 用户存取和 [工作区管理](../organization.md) 里的 HQ 工作区成员存取搞混。

## 打开此页面

打开 `POS Settings` -> `POS Users`。

## 本页控制内容

`POS Users` 会管理实体 POS 终端机使用的员工账号。

您可以在这里管理：

- 用户群组
- 每个用户群组的 POS 权限
- 单一 POS 用户纪录
- 登录方式
- 停用或启用中的用户存取

## 主要分区

### 用户群组

用户群组用于定义可重复使用的角色指派。每个群组也可以包含 POS 操作权限。

常见栏位：

- `Group ID`
- `Group Name`
- `Alt Name`
- `权限`

权限会分成以下区块：

- `点餐权限`
- `收银及报表`
- `管理及营运`

常见权限示例包括：

- `点餐`
- `功能菜单`
- `作废项目`
- `作废交易`
- `手动折扣`
- `开钱柜`
- `日结`
- `报表`
- `管理功能`
- `售罄管理`
- `入数 / 出数`
- `网上订单操作`
- `ODO 取餐订单`
- `POS 设置管理`

### POS 用户

单一用户会指派到特定门店。

常见栏位：

- `User Name`
- `Alt Name`
- `Password`
- `Staff Code`
- `Card No.`

### 登录行为

您可以为用户设置不同的登录方式，例如：

- 用户 ID 登录
- 卡片登录
- 员工代码登录

### 状态

使用停用或启用状态来控制该用户还能不能在 POS 上登录。

## 步骤

1. 打开 `POS Settings` -> `POS Users`。
2. 选择 `User Groups`。
3. 选择 `New User Group`，或编辑现有群组。
4. 输入 `Group Name`，并按需要输入 `Alt Name`。
5. 在 `权限` 勾选该群组可在 POS 执行的操作。
6. 保存群组。
7. 选择 `POS Users`。
8. 建立或打开目标 POS 用户。
9. 确认门店指派正确。
10. 输入登录和识别栏位。
11. 选择需要的用户群组。
12. 开启需要的登录方式。
13. 按需要把用户设为启用或停用。
14. 保存用户。

## 保存后的影响

这些更改会影响该员工账号在指派门店里如何登录 POS，以及可以完成哪些受保护的 POS 操作。

如果员工没有权限，POS 可能会显示 `权限不足`。如果操作需要授权，POS 可能会要求 `输入授权员工编号`。授权员工必须和第一位员工不同。

## 如何检查你的更改

1. 重新打开用户纪录，确认保存后的栏位正确。
2. 确认该用户已指派到正确门店。
3. 重新打开用户群组，确认已选择的 `权限`。
4. 有条件的话，在目标 POS 上验证预期的登录方式。
5. 使用正确员工账号测试一个受保护操作，例如 `日结`、`开钱柜`、`报表` 或 `POS 设置管理`。

## 如果出现问题

- 确认您编辑的是 POS 用户，而不是 HQ 工作区成员。
- 检查是否选错门店。
- 如果 POS 显示 `权限不足`，确认该员工所属群组已包含所需权限。
- 如果 POS 要求 `输入授权员工编号`，请经理或另一位已授权员工批准操作。
- 在再次更改用户识别栏位前，先检查登录方式开关。

## 何时需要找老板／管理员

- 该账号属于经理或其他较高权限角色。
- 您不确定该用户应该被停用、移转，还是重新建立。
- 您不确定某个群组是否应取得 `日结`、`开钱柜`、`入数 / 出数` 或 `POS 设置管理` 等经理级权限。

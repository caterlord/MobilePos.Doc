# 安全使用 MCP

:::info[适合对象]
任何通过 MCP 使用 AI 助理查看或更改 X1 HQ 数据的人员。
:::

## 开始之前

- 确认助理已连接官方 X1 MCP 服务器：`https://mcp.x1.tech/mcp`。
- 要求更改前，确认已选品牌、门店和渠道。
- 先使用只读请求。
- 不要在对话中粘贴密码、付款凭证、API key 或私人顾客信息。

## 安全更改模式

每次实时更改都按以下顺序进行：

1. 要求助理检查当前设置。
2. 要求提出建议更改。
3. 要求显示更改前后预览。
4. 检查受影响品牌、门店、渠道、品项和设置。
5. 只有当预览正确时才批准。
6. 如更改以后台任务执行，要求助理确认任务状态。
7. 在 HQ、POS、在线点餐或报表验证结果。

## 工具动作等级

| 等级 | 作用 | 使用方式 |
| :--- | :--- | :--- |
| 读取 | 只查询数据。 | 用于报表、设置检查和查找记录。 |
| 预览 | 准备建议更改。 | 每次写入实时数据前都要使用。 |
| 应用 | 应用已批准预览。 | 只在检查预览和范围后使用。 |

## 批准前要检查什么

菜单更改请检查：

- 品项名称
- 分类
- 价格
- 加配项
- 套餐规则
- 供应状态
- POS 菜单或在线点餐渠道

设置更改请检查：

- 品牌
- 门店
- 渠道
- 当前值
- 建议值
- 预期运营影响

## 好的批准提示

使用能要求助理显示范围和影响的提示：

- “批准前，显示每个受影响品项的表格。”
- “按分类和渠道分组显示更改。”
- “显示每项更改的当前值、建议值和原因。”
- “在我明确批准前，不要应用任何更改。”
- “应用后检查任务状态，并告诉我要在 HQ 检查什么。”

## 遇到以下情况请停止批准

- 助理无法显示当前值。
- 预览包含超出预期的品项或门店。
- 来源菜单不清楚。
- 助理正在猜测分类、加配项、套餐规则或价格。
- 你在营业时间更改实时价格。
- 你没有权限批准该更改。

## 如出现问题

- 在了解当前状态前，不要批准新的更改去修正第一个问题。
- 要求助理重新读取受影响记录。
- 将最新值与你批准的预览比较。
- 检查更改是否仍在后台任务中。
- 如更改影响销售、报表、付款、顾客点餐或实时服务，请升级处理。

## 何时要求店主/管理员协助

- 应用后更改的记录比预期多。
- 你无法验证结果。
- 助理要求不应分享的数据。
- 你需要撤回影响实时运营的更改。

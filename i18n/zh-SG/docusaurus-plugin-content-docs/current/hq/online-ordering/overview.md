:::info[适用对象]
负责维护 QR 点餐、App 点餐或第三方菜单路由等顾客端数字点餐渠道的管理员。
:::

## 开始前

- 确认正确的品牌，以及要调整的门店或渠道。
- 确认基础菜单与项目设置已经存在。
- 先判断您要变更的是分类显示、菜单分配、门店级设置、渠道可用性，还是顾客端文案。

## 本节用途

![线上点餐总览](/img/hq/online-ordering/oo_overview_page.png)

**线上点餐** 模块用于管理 POS 菜单目录如何投放到顾客端渠道，例如 BYOD（Bring Your Own Device）QR Code 点餐、原生移动应用，以及第三方 API 聚合平台。

它不会建立另一份独立的项目资料，而是直接与核心菜单目录同步，因此您可以选择性发布项目、套用不同价格层级，并只针对数字前台覆盖翻译内容。

## 如何使用本节

设置或检查某个渠道时，建议按以下顺序使用子页面：

1. [分类与项目](./categories-and-items.md)：决定哪些内容会显示在线上。
2. [线上菜单](./menus-and-combinations.md)：将线上项目整理成可用菜单。
3. [通用设置](./general-settings.md)：打开正确门店并管理门店级线上点餐设置。
4. [3rd Party Platforms](./third-party-platforms.md)：为单家门店订阅并配置外送或集成平台。
5. [渠道设置](./channel-mapping.md)：控制各订单渠道可售哪些分类。
6. [推广内容](./call-to-action.md)：管理加购提示。
7. [UI 翻译](./ui-translations.md)：调整顾客端文案。

## 常见任务

- 发布新的线上分类：先从 [分类与项目](./categories-and-items.md) 开始。
- 为单家门店启用外送或集成平台：使用 [3rd Party Platforms](./third-party-platforms.md)。
- 将分类路由到指定订单渠道：使用 [渠道设置](./channel-mapping.md)。
- 修改顾客端文案：使用 [UI 翻译](./ui-translations.md)。
- 调整订单规则等运营设置：使用 [通用设置](./general-settings.md)。

## 保存后的影响

线上点餐设置会直接影响顾客端渠道，因此保存后必须到目标渠道验证结果。即使 HQ 设置正确，如果您检查的是错误渠道或错误菜单，页面看起来仍可能不对。

## 如发生问题

- 先确认核心菜单目录中已经存在基础菜单资料。
- 确认您修改的是正确的渠道专用页面。
- 请在目标顾客流程中验证结果，不要只看 HQ。

## 何时请店主／管理员协助

- 变更会影响营业中的顾客端菜单。
- 您不确定问题是在基础菜单，还是在线上点餐设置。

:::info[适用对象]
负责维护 POS 员工登入纪录的管理员和经理。
:::

## 开始前

- 确认正确的品牌和目标店铺。
- 先决定您是要建立使用者群组，还是单一 POS 使用者。
- 不要把 POS 使用者存取和 [工作区管理](../organization.md) 里的 HQ 工作区成员存取搞混。

## 打开此页面

打开 `POS Settings` -> `POS Users`。

## 本页控制内容

`POS Users` 会管理实体 POS 终端机使用的员工帐号。

您可以在这里管理：

- 使用者群组
- 单一 POS 使用者纪录
- 登入方式
- 停用或启用中的使用者存取

## 主要分区

### 使用者群组

使用者群组用于定义可重复使用的角色指派。

常见栏位：

- 群组代码
- 群组名称
- 别名

### POS 使用者

单一使用者会指派到特定店铺。

常见栏位：

- `User Name`
- `Alt Name`
- `Password`
- `Staff Code`
- `Card No.`

### 登入行为

您可以为使用者设定不同的登入方式，例如：

- 使用者 ID 登入
- 卡片登入
- 员工代码登入

### 状态

使用停用或启用状态来控制该使用者还能不能在 POS 上登入。

## 步骤

1. 打开 `POS Settings` -> `POS Users`。
2. 建立或检视需要的使用者群组。
3. 建立或打开目标 POS 使用者。
4. 确认店铺指派正确。
5. 输入登入和识别栏位。
6. 开启需要的登入方式。
7. 按需要把使用者设为启用或停用。
8. 保存使用者。

## 保存后的影响

这些更改会影响该员工帐号在指派店铺里如何登入 POS。

## 如何检查你的更改

1. 重新打开使用者纪录，确认保存后的栏位正确。
2. 确认该使用者已指派到正确店铺。
3. 有条件的话，在目标 POS 上验证预期的登入方式。

## 如果出现问题

- 确认您编辑的是 POS 使用者，而不是 HQ 工作区成员。
- 检查是否选错店铺。
- 在再次更改使用者识别栏位前，先检查登入方式开关。

## 何时需要找老板／管理员

- 该帐号属于经理或其他较高权限角色。
- 您不确定该使用者应该被停用、移转，还是重新建立。

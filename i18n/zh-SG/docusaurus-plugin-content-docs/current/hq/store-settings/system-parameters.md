## 开始前

- 确认正确的品牌和店铺。
- 除非您了解目标参数和预期值，否则不要编辑本页。
- 更改任何内容前，先记录当前值。

## 打开此页面

打开 `Store Settings` -> `System Parameters`。

## 本页控制内容

`System Parameters` 存放会影响 POS 行为、列印，或与其他系统连接的进阶设定值。

本页风险很高。这里的更改可能会立即影响现场营运，或在下次同步／使用参数时才生效。

![System Parameters](/img/hq/store-settings/system_parameters_page.png)

:::danger[高风险页面]
在没有确认原因的情况下更改系统参数，可能会破坏设备连线、列印行为或报表逻辑。只有在您知道目标值，而且知道如何验证结果时，才应该使用本页。
:::

## 主要分区

### 锁定表格

页面会以锁定的唯读状态开启。

若要编辑：

1. 选择锁头或解除锁定控制。
2. 阅读解除锁定警告。
3. 确认解除锁定动作。

### 参数列

每一列都包含：

- `Code`
- `Description`
- `Value`
- `Enabled`

`Code` 用来识别参数。`Value` 是当前启用设定。`Enabled` 可在不删除资料的情况下，开启或关闭该参数。

### 单列保存行为

本页使用每一列独立的保存控制。只要您更改某一列，该列就会显示保存和重设动作。

请使用：

- 列内 `Save` 来保留该项更改
- 列内 `Reset` 来取消尚未保存的更改

### JSON 汇入与汇出

JSON 工具可以协助把参数从一间店铺复制到另一间店铺。

只有在您确定来源店铺和目标店铺应该使用相同进阶设定时，才应该使用这些工具。

## 步骤

1. 打开 `Store Settings` -> `System Parameters`。
2. 确认选定的店铺。
3. 解除编辑锁定。
4. 找到目标参数列。
5. 只更新您打算更改的栏位。
6. 保存该列。
7. 如果还有另一个已确认的参数要更新，再重复一次。

## 使用 JSON 复制参数

1. 打开来源店铺并汇出参数 JSON。
2. 切换到目标店铺。
3. 解除编辑锁定。
4. 汇入 JSON 档案。
5. 仔细检查差异。
6. 只选择您要套用的列。
7. 套用更改。

## 保存后的影响

这些更改会影响选定店铺的进阶营运行为。视参数而定，影响可能出现在列印、设备互动、报表或 POS 流程。

## 如何检查你的更改

1. 重新打开该列，确认数值已正确保存。
2. 到实际使用该参数的系统位置验证真实行为。
3. 如果您使用了 JSON 汇入，请在汇入完成后检视受影响的列。

## 如果出现问题

- 将受影响的列还原到先前确认的值。
- 在了解影响前，先停止继续更改其他参数。
- 如果问题来自 JSON 汇入，请先找出已更改的列，再重试。

## 何时需要找老板／管理员

- 您不知道正确的目标值。
- 该参数会影响付款、列印、设备通讯或报表逻辑。
- 问题是在汇入参数集之后出现，而且您无法安全地找出被更改的那一列。

:::info[适用对象]
负责日常门市作业的收银员与班次经理。
:::

## 开始前

- 终端在线
- 员工账号可用
- 打印机可正常使用

## SOP 检查清单

<Tabs>
  <TabItem value="opening" label="开店">
1. 登录并在启用时执行 `Clock In`。
2. 打开 `Devices` 并执行 `Test Print`。
3. 确认店铺名称与营业日期／时间正确。
4. 打开 `Ordering`，确认桌台画面可载入。
5. 先打开一次结账，确认付款方式可用。
  </TabItem>
  <TabItem value="during-shift" label="营业中">
1. 确认品项后尽快送单。
2. 打印机或网络报警后，先重新检查设备状态。
3. 只有客户要求时才使用分单付款。
4. 需要稽核记录时再加入备注。
5. 每个付款事件只指派一位负责人处理。
  </TabItem>
  <TabItem value="handover" label="交班">
1. 交班终端停止新增订单。
2. 完成，或清楚交接所有进行中的结账。
3. 确认待处理的作废／退款已经经理批准。
4. 把未解决事件连同交易编号交接出去。
  </TabItem>
  <TabItem value="closing" label="收班">
1. 处理完进行中订单与待处理付款。
2. 确认没有收银员卡在结账处理中。
3. 完成收支记录。
4. 确认经理已准备进行日结。
5. 执行日结并确认完成。
  </TabItem>
</Tabs>

## 什么叫做正常运作

- 班次开始时没有被打印机／付款问题卡住
- 每个事件都有清楚负责人
- 日结时没有未解决的结账

## 如果出现问题

:::warning[付款状态不清楚]
停止重复重试，立即找经理。
:::

:::warning[设备持续不可用]
先做一次基本检查，然后向经理回报，不要一直反复尝试。
:::

## 什么时候要找经理／管理员

- 需要更改 `POS Code` 或关键设定
- 退款／作废需要批准
- 日结警告或验证阻止收班

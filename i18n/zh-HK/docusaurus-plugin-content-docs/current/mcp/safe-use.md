# 安全使用 MCP

:::info[適合對象]
任何透過 MCP 使用 AI 助理查看或更改 X1 HQ 資料的人員。
:::

## 開始之前

- 確認助理已連接官方 X1 MCP 伺服器：`https://mcp.x1.tech/mcp`。
- 要求更改前，確認已選品牌、店舖及渠道。
- 先使用只讀要求。
- 不要在對話中貼上密碼、付款憑證、API key 或私人顧客資料。

## 安全更改模式

每次即時更改都按以下次序進行：

1. 要求助理檢查目前設定。
2. 要求提出建議更改。
3. 要求顯示更改前後預覽。
4. 檢查受影響品牌、店舖、渠道、品項及設定。
5. 只有當預覽正確時才批准。
6. 如更改以背景工作執行，要求助理確認工作狀態。
7. 在 HQ、POS、網上落單或報表驗證結果。

## 工具動作等級

| 等級 | 作用 | 使用方式 |
| :--- | :--- | :--- |
| 讀取 | 只查詢資料。 | 用於報表、設定檢查及尋找記錄。 |
| 預覽 | 準備建議更改。 | 每次寫入即時資料前都要使用。 |
| 套用 | 套用已批准預覽。 | 只在檢查預覽及範圍後使用。 |

## 批准前要檢查甚麼

菜單更改請檢查：

- 品項名稱
- 分類
- 價格
- 加配項
- 套餐規則
- 供應狀態
- POS 菜單或網上落單渠道

設定更改請檢查：

- 品牌
- 店舖
- 渠道
- 目前值
- 建議值
- 預期營運影響

## 好的批准提示

使用能要求助理顯示範圍及影響的提示：

- 「批准前，顯示每個受影響品項的表格。」
- 「按分類及渠道分組顯示更改。」
- 「顯示每項更改的目前值、建議值及原因。」
- 「在我明確批准前，不要套用任何更改。」
- 「套用後檢查工作狀態，並告訴我要在 HQ 檢查甚麼。」

## 遇到以下情況請停止批准

- 助理無法顯示目前值。
- 預覽包含超出預期的品項或店舖。
- 來源菜單不清楚。
- 助理正在猜測分類、加配項、套餐規則或價格。
- 你在營業時間更改即時價格。
- 你沒有權限批准該更改。

## 如出現問題

- 在了解目前狀態前，不要批准新的更改去修正第一個問題。
- 要求助理重新讀取受影響記錄。
- 將最新值與你批准的預覽比較。
- 檢查更改是否仍在背景工作中。
- 如更改影響銷售、報表、付款、顧客落單或即時服務，請升級處理。

## 何時要求店主/管理員協助

- 套用後更改的記錄比預期多。
- 你無法驗證結果。
- 助理要求不應分享的資料。
- 你需要撤回影響即時營運的更改。

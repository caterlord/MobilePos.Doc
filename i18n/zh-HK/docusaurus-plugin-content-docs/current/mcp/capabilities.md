# X1 MCP 可以做甚麼

:::info[適合對象]
想了解 AI 助理可透過 X1 MCP 協助哪些 HQ 工作的店主、經理及管理員。
:::

## 開始之前

- 將 AI 助理連接到 X1 MCP。
- 確認助理能列出正確品牌及店舖。
- 在要求更改前，先使用只讀問題。

## 核心概念

X1 MCP 將 HQ 工作變成受引導的助理流程：

1. 檢查目前設定。
2. 準備建議更改。
3. 顯示更改前後預覽。
4. 等待批准。
5. 套用已批准更改。
6. 驗證結果。

## 報表及分析

不用逐個開啟 HQ 報表，也可以要求助理檢查表現。

例子：

- 「按店舖顯示最近 7 日銷售。」
- 「比較今個星期一與上星期一。」
- 「今星期哪些分類增長最多？」
- 「顯示熱飲品項表現。」
- 「找出需要注意的營運異常。」

助理可以整理銷售、趨勢、期間比較、品項表現、分類表現及營運異常。

## 菜單建立及匯入

當你有試算表、PDF、圖片或文字菜單來源時使用。

建議要求：

> 讀取這份菜單，先顯示分類、品項、價格、加配項及套餐規則，不要立即建立任何 X1 資料。

助理可以協助：

- 讀取現有 X1 菜單結構
- 準備結構化菜單草稿
- 驗證分類、品項、加配項及套餐規則
- 預覽菜單匯入
- 套用已批准匯入

請仔細檢查預覽。上載菜單常有縮寫、手寫備註或價格規則，需要人工確認。

## 菜單維護

用於更改現有目錄。

例子：

- 「將所有熱飲加價 `$1`。」
- 「今天將這些品項設為網上落單不可售。」
- 「將這些品項由 `Lunch` 移到 `Dinner`。」
- 「比較兩間店的菜單設定。」
- 「顯示差異後，將一間店的菜單設定複製到另一間店。」

助理可以讀取品項、分類、加配項、智能分類及 POS 菜單。它可以預覽及套用品項更改、價格調整、供應狀態、重新分類及指定批量菜單更新。

## 加配項及套餐規則

當品項需要選項、加購或套餐規則時使用。

例子：

- 「為咖啡建立牛奶選項加配組。」
- 「建立套餐規則：任何 `Lunch` 品項包一杯熱飲，轉凍飲加 `$3`。」
- 「更改前顯示哪些品項正在使用這個加配組。」

助理可以在你批准後預覽及套用加配組更改和套餐規則更改。

## 網上落單準備狀態

發布或排查顧客可見菜單前使用。

例子：

- 「檢查這間店是否已準備好網上落單。」
- 「為甚麼這個品項未在網上顯示？」
- 「顯示缺少的分類、菜單或渠道設定。」

助理可以檢查網上落單設定、診斷菜單顯示、檢查發布準備狀態，並預覽指定網上落單設定更改。

## 店舖及裝置設定

用於檢查及小範圍受控更改。

例子：

- 「顯示這間店的店舖設定快照。」
- 「顯示裝置設定及終端設定。」
- 「先顯示現有名稱，再重新命名這個錢箱。」

助理可以檢查店舖設定、裝置設定，並預覽指定裝置設定更改，例如錢箱名稱。

## 背景工作狀態

部分已批准更改可能以背景工作執行。

你可以問：

> 檢查工作狀態，並告訴我更改是否已成功完成。

在 HQ、POS 或網上落單驗證結果前，先完成這一步。

## 何時要求店主/管理員協助

- 助理建議跨多間店更改。
- 更改影響即時價格、菜單顯示、網上落單、付款相關設定或裝置。
- 預覽沒有清楚顯示所有受影響記錄。
- 來源菜單不清楚，或與目前 HQ 設定有衝突。

:::info[適用對象]
管理進階店舖行為及技術營運參數的店主／管理員用戶。
:::

## 開始前

- 確認正確品牌及店舖。
- 除非你清楚目標參數及預期值，否則不要編輯此頁。
- 更改任何內容前，先記錄目前值。

## 開啟此頁

前往 `店舖設定` -> `系統參數`。

## 本頁控制內容

`系統參數` 儲存一些進階設定值，這些值可能會影響 POS 的行為、列印，或與其他系統的連接方式。

這一頁屬於高風險頁面。變更可能會即時生效，或在下次同步／使用該參數時生效。

![系統參數](/img/hq/store-settings/system_parameters_page.png)

:::danger[高風險頁面]
如果沒有確認原因就更改系統參數，可能會破壞裝置連接、列印行為或報表邏輯。只有在你已知目標值，而且知道如何驗證結果時，才應使用此頁。
:::

## 主要區塊

### 鎖定表格

頁面預設以鎖定、唯讀狀態開啟。

如要編輯：

1. 選擇鎖頭或解鎖控制。
2. 查看解鎖警告。
3. 確認解鎖動作。

### 參數列

每一列都包含：

- `Code`
- `Description`
- `Value`
- `Enabled`

`Code` 用來識別參數。`Value` 是啟用中的設定。`Enabled` 會控制參數是否生效，但不會刪除它。

### 有變更列的儲存行為

本頁使用列級儲存控制。當你更改某一列時，該列會顯示儲存及重設動作。

你可以使用：

- 該列的 `Save` 來保留變更
- 該列的 `Reset` 來取消未儲存變更

### JSON 匯入與匯出

JSON 工具可協助你把參數從一間店舖複製到另一間。

只有在你確定來源及目標店舖應該共用同一組進階設定時，才應使用。

## 步驟

1. 前往 `店舖設定` -> `系統參數`。
2. 確認選定的店舖。
3. 解鎖編輯。
4. 找到目標參數列。
5. 只修改你打算更改的欄位。
6. 儲存該列。
7. 只有在你還有另一個已確認參數要改時，才重複操作。

## 透過 JSON 複製參數

1. 開啟來源店舖並匯出參數 JSON。
2. 切換到目標店舖。
3. 解鎖編輯。
4. 匯入 JSON 檔案。
5. 仔細檢查差異。
6. 只選擇你要套用的列。
7. 套用變更。

## 儲存後的影響

這些變更會影響選定店舖的進階營運行為。視乎參數而定，效果可能出現在列印、裝置互動、報表或 POS 流程。

## 如何確認變更

1. 重新開啟該列，確認值已正確儲存。
2. 在真正使用該參數的區域驗證實際行為。
3. 如果你用了 JSON 匯入，請在匯入完成後檢查受影響的各列。

## 如果出現問題

- 回復受影響列到之前已確認的值。
- 在你理解影響之前，先停止再做其他參數變更。
- 如果問題來自 JSON 匯入，先找出被改動的列，再嘗試重做。

## 何時要找店主／管理員

- 你不知道正確目標值。
- 這項參數影響付款、列印、裝置通訊或報表邏輯。
- 匯入參數集後出現問題，而你無法安全地分辨是哪一列被改動。

## 開始前

- 確認正確的品牌，以及要設定的店舖或渠道。
- 確認基礎菜單與項目已經建立完成。
- 先判斷您要變更的是分類顯示、菜單指派、渠道路由，還是顧客端文字。

## 本節用途

![網上點餐總覽](/img/hq/online-ordering/oo_overview_page.png)

**網上點餐** 模組會管理 POS 菜單目錄如何發佈到顧客端渠道，例如 BYOD（Bring Your Own Device）QR Code 點餐、原生行動應用程式，以及第三方 API 聚合平台。

它不會另外建立一份獨立的項目資料庫，而是直接與核心菜單目錄同步，因此您可以選擇性發佈項目、套用不同的價格層級，並針對數位商店覆寫翻譯內容。

## 如何使用本節

設定或檢視渠道時，請依照下列順序使用子頁面：

1. [分類及項目](./categories-and-items.md)：決定哪些內容會顯示在網上。
2. [網上菜單](./menus-and-combinations.md)：將網上項目分組成可使用的菜單。
3. [渠道對應](./channel-mapping.md)：將菜單路由到正確的訂單渠道。
4. [一般設定](./general-settings.md)：套用給指定店舖或渠道的營運控制。
5. [推廣內容](./call-to-action.md)：管理加購提示。
6. [UI 翻譯](./ui-translations.md)：調整顧客端文字。

## 常見工作

- 發佈新的網上分類：先從 [分類及項目](./categories-and-items.md) 開始。
- 將菜單路由到特定訂單渠道：使用 [渠道對應](./channel-mapping.md)。
- 變更顧客端文字：使用 [UI 翻譯](./ui-translations.md)。
- 調整訂單規則等營運設定：使用 [一般設定](./general-settings.md)。

## 儲存後的影響

網上點餐的變更會影響顧客端渠道，所以儲存後要到目標渠道確認。即使 HQ 設定正確，如果檢查的是錯誤的渠道或菜單，畫面也可能看起來不對。

## 如發生問題

- 先確認核心菜單目錄中已存在基礎菜單資料。
- 確認您修改的是正確的渠道專屬頁面。
- 請在目標顧客流程中驗證結果，不要只看 HQ。

## 何時要請店主／管理員協助

- 變更會影響營業中的顧客端菜單。
- 您不確定問題是在基礎菜單，還是在網上點餐設定。

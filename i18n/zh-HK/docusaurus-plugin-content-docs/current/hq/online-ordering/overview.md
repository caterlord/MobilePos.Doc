## 開始前

- 確認正確的品牌，以及要調整的店舖或渠道。
- 確認基礎菜單與項目設定已經存在。
- 先判斷您要變更的是分類顯示、菜單指派、店舖層級設定、渠道可用性，還是顧客端文字。

## 本節用途

![網上點餐總覽](/img/hq/online-ordering/oo_overview_page.png)

**網上點餐** 模組會管理 POS 菜單目錄如何投放到顧客端渠道，例如 BYOD（Bring Your Own Device）QR Code 點餐、原生流動應用程式，以及第三方 API 聚合平台。

它不會建立另一份獨立的項目資料，而是直接與核心菜單目錄同步，因此您可以選擇性發佈項目、套用不同價格層級，並只針對數位前台覆寫翻譯內容。

## 如何使用本節

設定或檢查某個渠道時，建議依照以下順序使用子頁面：

1. [分類及項目](./categories-and-items.md)：決定哪些內容會顯示在網上。
2. [網上菜單](./menus-and-combinations.md)：將網上項目整理成可用菜單。
3. [一般設定](./general-settings.md)：開啟正確店舖並管理店舖層級網上點餐設定。
4. [3rd Party Platforms](./third-party-platforms.md)：為單一店舖訂閱與設定外送或整合平台。
5. [渠道設定](./channel-mapping.md)：控制各個訂單渠道可售哪些分類。
6. [推廣內容](./call-to-action.md)：管理加購提示。
7. [UI 翻譯](./ui-translations.md)：調整顧客端文案。

## 常見工作

- 發佈新的網上分類：先從 [分類及項目](./categories-and-items.md) 開始。
- 為單一店舖啟用外送或整合平台：使用 [3rd Party Platforms](./third-party-platforms.md)。
- 把分類路由到指定訂單渠道：使用 [渠道設定](./channel-mapping.md)。
- 變更顧客端文案：使用 [UI 翻譯](./ui-translations.md)。
- 調整訂單規則等營運設定：使用 [一般設定](./general-settings.md)。

## 儲存後的影響

網上點餐設定會直接影響顧客端渠道，所以儲存後必須到目標渠道驗證結果。即使 HQ 設定正確，如果您檢查的是錯誤渠道或錯誤菜單，畫面仍可能看起來不對。

## 如發生問題

- 先確認核心菜單目錄中已經有基礎菜單資料。
- 確認您修改的是正確的渠道專屬頁面。
- 請在目標顧客流程中驗證結果，不要只看 HQ。

## 何時要請店主／管理員協助

- 變更會影響營業中的顧客端菜單。
- 您不確定問題是在基礎菜單，還是在網上點餐設定。

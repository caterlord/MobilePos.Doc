:::info[適用對象]
維護作廢、收款入賬及支出等受控 POS 動作標準原因清單的管理員。
:::

## 開始前

- 確認正確品牌。
- 決定哪一種營運動作需要原因清單。
- 把清單保持簡短清晰，方便員工在 POS 上快速使用。

## 開啟此頁

前往 `POS 設定` -> `原因代碼`。

## 本頁控制內容

`原因代碼` 定義員工執行受控動作時要選取的標準解釋。

這樣可以避免自由輸入原因，並保持稽核報表一致。

![原因代碼頁面](/img/hq/pos-settings/function_reasons_page.png)

## 主要區塊

### 原因群組

常見群組包括：

1. `TX_DISABLE`
2. `PAYIN`
3. `PAYOUT`
4. `ISO`

每個群組都對應不同的 POS 動作類型。

### POS 行為

當某個 POS 動作需要原因時，終端機會顯示對應群組中可用及已啟用的原因，供員工選擇。

## 步驟

1. 前往 `POS 設定` -> `原因代碼`。
2. 選擇正確的原因群組。
3. 新增或編輯原因文字。
4. 儲存原因。
5. 只有在其他群組也需要更新時，才重複操作。

## 儲存後的影響

這些變更會影響員工在相關 POS 動作需要原因時可以選擇哪些項目。

## 如何確認變更

1. 重新開啟清單，確認原因位於正確群組。
2. 確認文字簡短、清晰，適合員工快速選取。
3. 在可行時，測試相關 POS 動作，確認原因會在預期位置出現。

## 如果出現問題

- 確認原因已分配到正確群組。
- 檢查該 POS 動作是否本來就設定為需要原因，不要先假設清單失效。
- 如果報表看起來不一致，先檢查是否有重複或不清晰的原因名稱。

## 何時要找店主／管理員

- 這份原因清單會影響合規、損耗預防或稽核報表。
- 你不確定某個原因應該屬於哪個動作群組。

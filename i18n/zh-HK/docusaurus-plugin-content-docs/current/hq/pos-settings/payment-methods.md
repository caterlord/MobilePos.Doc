:::info[適用對象]
定義收銀員在結帳時可使用哪些付款方式，以及各付款方式行為的管理員。
:::

## 開始前

- 確認正確品牌。
- 決定你要建立的是基本付款方式，還是要連結閘道的付款方式。
- 如果該方式依賴供應商，先確認供應商已在 [付款供應商](./payment-vendors.md) 完成設定。

## 開啟此頁

前往 `POS 設定` -> `付款方式`。

## 本頁控制內容

`付款方式` 定義 POS 可用的付款按鈕及結帳行為。

你可以在這一頁控制：

- 付款名稱與按鈕順序
- 已連結閘道的行為
- 附加費及收費率設定
- 現金錢箱與找零行為
- 每店舖啟用

這個設定通常先在品牌層級建立，再於店舖層級啟用或配置。

![付款方式頁面](/img/hq/pos-settings/pos_payment_methods_page.png)

## 主要區塊

### 基本欄位

| 欄位 | 控制內容 | 何時更改 |
| --- | --- | --- |
| `Code` 及 `Name` | 內部及 POS 顯示的付款識別 | 新增或改名付款方式時 |
| `Display Index` | 結帳時按鈕排序 | 收銀員需要不同付款順序時 |
| `Linked Gateway` | 此付款方式使用的付款閘道或裝置驅動 | 該方式需要連接外部終端機或服務時 |
| `Surcharge Rate` | 付款時加到客戶端的額外收費 | 該方式需要附加費時 |
| `Tx Charges Rate` | 供財務追蹤的商戶費用 | 財務需要看到費用時 |
| `Fixed Amount` | 直接以預設金額結帳 | 使用固定金額券類付款時 |

### POS 行為開關

這些開關會控制員工點選付款方式時 POS 的反應：

- `Open Cash Drawer`
- `Allow Tips`
- `Non-Sales Payment`
- `Cash Payment`
- `Over Payment`
- `FX Payment`
- `Auto Remark`

### 店舖設定

基本付款方式建立後，請到店舖設定區域，僅為需要使用的店舖啟用。這是使用者最常漏做的步驟。

## 步驟

1. 前往 `POS 設定` -> `付款方式`。
2. 新增付款方式，或開啟現有付款方式。
3. 輸入 `Code`、`Name` 及 `Display Index`。
4. 只有當此付款方式需要連接付款供應商時，才選擇 `Linked Gateway`。
5. 視需要設定費率與行為開關。
6. 打開店舖設定區域。
7. 為目標店舖啟用該方式。
8. 儲存付款方式。

## 已連結閘道

如果該付款方式依賴第三方或本地付款供應商，請先在 [付款供應商](./payment-vendors.md) 完成供應商設定。

:::warning[供應商依賴]
不要連結品牌未安裝或未配置的閘道。沒有正常供應商設定的閘道付款方式，結帳時可能會失敗。
:::

## 儲存後的影響

儲存基本付款方式會建立或更新該付款定義。不過，員工要真正使用它之前，仍然要先為正確店舖啟用。

## 如何確認變更

1. 重新開啟付款方式，確認基本欄位已正確儲存。
2. 確認該方式已為目標店舖啟用。
3. 如果該方式是閘道連結型，請在 [付款供應商](./payment-vendors.md) 再核對相關供應商設定。
4. 在可行時，於目標店舖的結帳流程中測試該付款方式。

## 如果出現問題

- 確認問題不是出在店舖啟用步驟。
- 檢查是否連結了錯誤或缺少的閘道。
- 如果附加費或錢箱行為不正確，在改動其他欄位前，先只檢查受影響的開關。

## 何時要找店主／管理員

- 這項變更會影響即時付款流程。
- 你不確定是否需要新增供應商或連結閘道。
- 這個付款方式會影響財務、附加費政策或多店舖推行。

:::info[適用對象]
維護 POS 員工登入記錄的管理員和經理。
:::

## 開始前

- 確認正確品牌及目標店舖。
- 決定你要建立使用者群組，還是個別 POS 員工。
- 決定該群組可在 POS 執行哪些操作。
- 不要把 POS 員工存取與 HQ 工作區成員存取混淆，後者可在 [工作區管理](../organization.md) 查看。

## 開啟此頁

前往 `POS 設定` -> `POS 員工`。

## 本頁控制內容

`POS 員工` 管理實體 POS 終端機上使用的員工帳號。

你可以在這一頁管理：

- 使用者群組
- 每個使用者群組的 POS 權限
- 個別 POS 員工記錄
- 登入方式
- 啟用或停用的使用者存取

## 主要區塊

### 使用者群組

使用者群組用來定義可重用的 POS 員工角色。每個群組亦可包含 POS 操作權限。

常見欄位：

- `Group ID`
- `Group Name`
- `Alt Name`
- `權限`

權限會分成以下區塊：

- `點餐權限`
- `收銀及報表`
- `管理及營運`

常見權限例子包括：

- `點餐`
- `功能選單`
- `作廢項目`
- `作廢交易`
- `手動折扣`
- `開錢櫃`
- `日結`
- `報表`
- `管理功能`
- `售罄管理`
- `入數 / 出數`
- `網上訂單操作`
- `ODO 取餐訂單`
- `POS 設定管理`

### POS 員工

個別使用者會指派到特定店舖。

常見欄位：

- `User Name`
- `Alt Name`
- `Password`
- `Staff Code`
- `Card No.`

### 登入行為

使用者可以配置不同登入方式，例如：

- 使用者 ID 登入
- 卡片登入
- 員工代碼登入

### 狀態

使用停用或啟用狀態，控制該使用者是否仍可在 POS 上登入。

## 步驟

1. 前往 `POS 設定` -> `POS 員工`。
2. 選擇 `User Groups`。
3. 選擇 `New User Group`，或編輯現有群組。
4. 輸入 `Group Name`，並按需要輸入 `Alt Name`。
5. 在 `權限` 勾選此群組可在 POS 執行的操作。
6. 儲存群組。
7. 選擇 `POS Users`。
8. 建立或開啟目標 POS 員工。
9. 確認店舖指派正確。
10. 輸入登入及識別欄位。
11. 選擇所需的使用者群組。
12. 開啟所需的登入方式。
13. 視需要把使用者設為啟用或停用。
14. 儲存使用者。

## 儲存後的影響

這些變更會影響已指派店舖中，該員工帳號如何登入 POS，以及可以完成哪些受保護的 POS 操作。

如果員工沒有權限，POS 可能會顯示 `權限不足`。如果操作需要授權，POS 可能會要求 `輸入授權員工編號`。授權員工必須與第一位員工不同。

## 如何確認變更

1. 重新開啟使用者記錄，確認已儲存的欄位。
2. 確認使用者已指派到正確店舖。
3. 重新開啟使用者群組，確認已選擇的 `權限`。
4. 在可行時，於目標 POS 測試所需登入方式。
5. 使用正確員工帳號測試一個受保護操作，例如 `日結`、`開錢櫃`、`報表` 或 `POS 設定管理`。

## 如果出現問題

- 確認你編輯的是 POS 員工，而不是 HQ 工作區成員。
- 檢查是否選錯店舖。
- 如果 POS 顯示 `權限不足`，確認該員工所屬群組已包含所需權限。
- 如果 POS 要求 `輸入授權員工編號`，請經理或另一位已授權員工批准操作。
- 在再次更改使用者識別資料前，先檢查登入方式開關。

## 何時要找店主／管理員

- 這個帳號屬於經理或其他權限較高的角色。
- 你不確定該使用者應該停用、搬移，還是重新建立。
- 你不確定某個群組是否應取得 `日結`、`開錢櫃`、`入數 / 出數` 或 `POS 設定管理` 等經理級權限。

:::info[適用對象]
負責建立、維護或追蹤 POS 中可售、可客製化或可被引用商品的管理員。
:::

## 開始前

- 確認正確的品牌。
- 先判斷該記錄是可售項目、改碼項目，還是套餐項目。
- 儲存前先確認目標分類與部門。
- 如果要複製商品，請先準備未被啟用商品使用的全新 `Item Code`。

## 開啟此頁面

開啟 `Menu Management` -> `Menu Items`。

## 此頁控制內容

`Menu Items` 是 POS 菜單的主要商品資料庫。

可用來建立或維護：

- 可售商品
- 改碼項目
- 套餐或組合的父項目
- 商品價格與供應狀態
- 特殊商品行為
- 以現有商品設定建立的新商品複本

![菜單項目清單](/img/hq/menu-catalog/menu_items_list.png)

## 主要分區

### 商品類型

先選擇正確的商品類型，再繼續操作：

- 可售商品
- 改碼
- 套餐項目

商品類型會影響商品歸屬，以及 POS 要如何使用它。

### 基本欄位

| 欄位 | 控制內容 | 何時要調整 |
| --- | --- | --- |
| `商品代碼` | 內部商品識別碼 | 當 SKU 或整合代碼改變時 |
| `商品名稱` | HQ 的主要名稱 | 當核心記錄名稱改變時 |
| `POS 顯示名稱` | 收銀員在 POS 上看到的名稱 | 當販售名稱要與 HQ 名稱不同時 |
| `POS 列印名稱` | 收據或廚房單據上列印的名稱 | 當需要更短或更適合營運的文字時 |
| 分類與部門欄位 | 瀏覽結構與報表結構 | 當商品需要歸到不同菜單位置或彙總層級時 |

### 特殊行為開關

請小心使用商品開關，因為它們可能會改變結帳時的 POS 行為。

例如：

- 手動價格
- 不可折扣
- 不收服務費
- 不計稅
- 單據顯示選項

### 價格與供應狀態

使用商品的價格與供應區塊，控制分店專屬售價，以及商品是否啟用或已售罄。

### 複製現有商品

當新商品應從現有商品設定開始建立時，請使用複製動作。

1. 找到要複製的商品。
2. 在 `Actions` 欄選擇複製動作。
3. 輸入新的 `Item Code`、`Item Name`、`Item Name (Alt)`、`Item Kitchen Name` 及 `Item Kitchen Name (Alt)`。
4. 確認目標 `Category` 與 `Department`。
5. 儲存複本。

複製會帶入來源商品的相關設定，例如店舖資料、價格、改碼組別、套餐群組及其他商品關係。

### 猛獸模式複製

如果需要從同一個來源商品一次建立多個新商品，請在複製視窗開啟 `Beast Mode`。

開啟 `Beast Mode` 後，每一列代表一個目標商品：

- `Item Code`
- `Item Name`
- `Item Name (Alt)`
- `Item Kitchen Name`
- `Item Kitchen Name (Alt)`

同一次儲存中的所有列會共用同一個目標 `Category` 與 `Department`。

儲存前，請確認每個 `Item Code` 在列清單中不重複，且沒有被任何啟用中的商品使用。只要其中一個目標商品無法儲存，本次所有目標商品都不會建立。

## 步驟

1. 開啟 `Menu Management` -> `Menu Items`。
2. 新增商品或開啟現有商品。
3. 選擇正確的商品類型。
4. 輸入商品識別欄位。
5. 將商品指派到正確的分類與部門。
6. 套用需要的特殊行為開關。
7. 檢查目標店舖的價格與供應狀態。
8. 儲存商品。

## 儲存後的影響

這些變更會影響共用的商品記錄，也可能影響店舖可用性、POS 瀏覽、報表，以及結帳行為，視您實際修改的內容而定。

## 如何檢查你的變更

1. 重新開啟商品，確認儲存的值正確。
2. 驗證商品是否掛在正確的分類與部門底下。
3. 如果有調整店舖價格或供應狀態，請再確認目標店舖設定。
4. 如果您修改了特殊開關，請實際測試受影響的 POS 行為。
5. 如果您複製了商品，請重新開啟複製後的商品，確認相關設定已按預期複製。

## 如發生問題

- 確認錯誤結果不是由分類、部門或供應狀態造成。
- 如果 POS 行為不符合預期，先檢查商品類型。
- 如果商品存在但某一家店舖看不到，請檢查該店舖的價格或供應設定。
- 如果複製失敗，請檢查複製清單中或啟用中的商品是否已有相同的 `Item Code`。

## 何時要請店主／管理員協助

- 變更會影響正式價格或顧客可見的列印內容。
- 您不確定商品應該是可售商品、改碼，還是套餐項目。

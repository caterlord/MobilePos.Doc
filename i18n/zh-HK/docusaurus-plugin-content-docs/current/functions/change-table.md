---
title: 變更桌台
---

:::info[適用對象]
需要把未完成訂單移到另一張桌台的收銀員和更班經理。
:::

## 開始前

- 您已登入並位於訂單畫面。
- 您已開啟正確的訂單。
- 您可以打開功能選單（三點按鈕）。
- 您已按 `Send` 送出尚未提交的品項修改。
- 您已確認目標桌台是空桌。

## 步驟

1. 按功能選單（三點按鈕）。
2. 在 `Select Function` 中按 `Change Table`。
3. 如果 `Change Table` 在其他頁，按 `Previous` 或 `Next`。
4. 在 `Change Table` 畫面，如需切換 `Table List` 和 `Floor Plan`，可按右下角切換按鈕。
5. 如需顯示分桌，按 `Show Split`。
6. 按目標桌台。
7. 等待桌台選擇器關閉。
8. 如果應用程式返回桌台清單，請從目標桌台重新開啟已移動的訂單。

## 您會看到

- 訂單不再顯示在原本桌台。
- 訂單會出現在目標桌台。
- 您可以在目標桌台繼續點餐。
- 啟用平面圖模式時，右下角切換按鈕可在 `Table List` 和 `Floor Plan` 之間切換。
- 顯示分桌後，`Show Split` 會變成 `Hide Split`。

## 如果出現問題

- 如果看到 `No item to change table`，請先加入至少一個品項。
- 如果看到 `Cannot change table` 並顯示 `Please save all operations before changing table`，先按 `Send`，再重試。
- 如果在 `Floor Plan` 找不到目標桌台，請切換到 `Table List`。
- 如果目標分桌未出現，請按 `Show Split`。
- 如果看到 `Please select an empty table`，請改選其他空桌。
- 如果看到 `Unable to change table`，先重新整理訂單，再重試一次。

## 何時需要找經理／管理員

- `Manager only`：如屬特別處理或爭議訂單才可移桌。收銀員可先維持原桌並繼續服務。
- 如無可用的目標桌台，請找經理。
- 如果 `Change Table` 重試一次後仍失敗，請找管理員。

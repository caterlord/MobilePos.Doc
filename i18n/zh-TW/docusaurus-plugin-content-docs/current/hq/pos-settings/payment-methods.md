:::info[適用對象]
適合定義收銀員在結帳時可使用哪些付款方式，以及這些方式如何運作的管理員。
:::

## 開始前

- 確認你目前選到正確的品牌。
- 先決定你要建立的是基本付款方式，還是連結閘道的付款方式。
- 如果這個方式依賴供應商，先確認該供應商已在 [付款供應商](./payment-vendors.md) 中設定完成。

## 開啟此頁面

開啟 `POS Settings` -> `Payment Methods`。

## 此頁控制內容

`Payment Methods` 定義 POS 可用的付款按鈕與結帳行為。

本頁可控制：

- 付款名稱與按鈕順序
- 連結閘道行為
- 附加費率與收費率
- 現金抽屜與找零行為
- 每家店舖的啟用狀態

這類設定通常先在品牌層級建立，再依店舖啟用或調整。

![付款方式頁面](/img/hq/pos-settings/pos_payment_methods_page.png)

## 主要分區

### 基本欄位

| 欄位 | 控制內容 | 何時變更 |
| --- | --- | --- |
| `Code` 與 `Name` | 付款方式的內部與 POS 顯示識別 | 新增或重新命名付款方式時 |
| `Display Index` | 結帳時的按鈕順序 | 需要調整收銀員的操作順序時 |
| `Linked Gateway` | 這個付款方式使用的付款閘道或裝置驅動 | 付款方式需要連接外部終端機或服務時 |
| `Surcharge Rate` | 付款時加收的對客附加費 | 該付款方式需要附加費時 |
| `Tx Charges Rate` | 只供報表用的商戶手續費追蹤 | 財務需要看手續費時 |
| `Fixed Amount` | 以預設金額直接結帳 | 使用固定金額禮券式付款方式時 |

### POS 行為開關

這些開關會決定員工點選付款方式時，POS 要如何回應：

- `Open Cash Drawer`
- `Allow Tips`
- `Non-Sales Payment`
- `Cash Payment`
- `Over Payment`
- `FX Payment`
- `Auto Remark`

### 店舖設定

完成基本付款方式後，請到店舖設定區啟用它，讓需要使用的店舖可以看到。這是最常被漏掉的一步。

## 步驟

1. 開啟 `POS Settings` -> `Payment Methods`。
2. 建立新的付款方式，或開啟既有付款方式。
3. 輸入 `Code`、`Name` 與 `Display Index`。
4. 只有在這個付款方式需要連到付款供應商時，才選擇 `Linked Gateway`。
5. 視需要設定費率與行為開關。
6. 開啟店舖設定區。
7. 為目標店舖啟用這個付款方式。
8. 儲存付款方式。

## 連結閘道

如果這個付款方式依賴第三方或場內付款供應商，請先到 [付款供應商](./payment-vendors.md) 設定供應商。

:::warning[供應商依賴]
不要連結品牌尚未安裝或未完成設定的閘道。如果付款方式連到的閘道沒有正常可用的供應商設定，結帳時可能會失敗。
:::

## 儲存後的影響

儲存基本付款方式會建立或更新付款定義，但員工要能在正確店舖中實際使用它，還是必須先啟用。

## 如何檢查你的變更

1. 重新開啟該付款方式，確認基本欄位已正確儲存。
2. 確認這個方式已啟用給預定的店舖。
3. 如果這個方式是連結閘道，請到 [付款供應商](./payment-vendors.md) 驗證相關設定。
4. 在實際可行時，於目標店舖的結帳流程測試這個付款方式。

## 如發生問題

- 先確認問題不是出在店舖啟用步驟。
- 檢查是否連到了錯誤或缺少的閘道。
- 如果附加費或現金抽屜行為錯誤，先只檢查受影響的開關，不要先改其他欄位。

## 何時要請店主／管理員協助

- 這次變更會影響正式付款流程。
- 你不確定是否需要新增供應商或連結閘道。
- 這個付款方式會影響財務、附加費政策或多店鋪佈署。

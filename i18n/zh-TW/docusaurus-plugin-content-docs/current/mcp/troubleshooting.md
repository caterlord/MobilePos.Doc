# MCP 疑難排解

:::info[適合對象]
需要修正 MCP 連接、登入、權限、預覽、套用或驗證問題的管理員。
:::

## 開始之前

- 確認 AI 工具使用 `https://mcp.x1.tech/mcp`。
- 確認你可以用正確 X1 帳戶登入。
- 確認你知道目標品牌及店鋪。
- 疑難排解期間避免批准寫入動作。

## 助理無法連接

1. 確認伺服器網址完全是 `https://mcp.x1.tech/mcp`。
2. 確認工具支援遠端 HTTPS MCP 伺服器。
3. 在 AI 工具中重新連接 MCP 伺服器。
4. 如工具要求授權，請重新登入。
5. 嘗試只讀提示：「列出我可存取的品牌及店鋪。」

在瀏覽器開啟 `https://mcp.x1.tech/mcp` 可能會顯示 method error。這不一定代表伺服器故障，因為 MCP 工具使用端點的方式與瀏覽器頁面不同。

## 登入或授權失敗

1. 確認你使用正確 X1 帳戶。
2. 確認彈出視窗或外部登入視窗未被封鎖。
3. 登出 AI 工具後重新登入。
4. 要求店主/管理員確認你的 X1 角色可存取目標品牌及店鋪。

## 助理看不到正確品牌或店鋪

1. 詢問：「列出我可存取的所有品牌及店鋪。」
2. 確認缺少的店鋪是否屬於其他品牌或工作區。
3. 要求店主/管理員檢查你的 HQ 存取權限。
4. 權限更改後重新連接 MCP。

## 助理建議錯誤範圍

批准前停止。

要求助理顯示：

- 已選品牌
- 已選店鋪
- 已選渠道
- 受影響分類、品項或設定
- 每項記錄被包含的原因

修正範圍後，要求重新預覽。

## 預覽看起來不正確

1. 不要批准套用。
2. 要求助理顯示所使用的來源資料。
3. 修正品項名稱、分類、加配項、價格或套餐規則。
4. 要求重新預覽。
5. 將新預覽與來源檔案或目前 HQ 設定比較。

## 已套用更改但看不到結果

1. 要求助理檢查工作狀態。
2. 重新整理 HQ 頁面。
3. 確認你正在查看被更改的同一品牌、店鋪及渠道。
4. 檢查使用該資料的位置：POS、線上點餐、報表或目標店鋪。
5. 要求助理重新讀取受影響記錄。

## 上傳菜單匯入不清楚

要求助理暫停，並先顯示解析出的資料。

檢查：

- 分類名稱
- 品項名稱
- 價格欄
- 必選加配項
- 可選加配項
- 套餐或餐組規則
- 可能不屬於菜單的備註

只有當解析結構正確時，才要求預覽。

## 何時要求店主/管理員協助

- 你無法登入或授權 MCP 連接。
- 你的帳戶看不到預期品牌或店鋪。
- 預覽包含非預期店鋪、渠道或記錄。
- 套用完成但即時結果不正確。
- 問題影響營業、顧客點餐、付款或報表。

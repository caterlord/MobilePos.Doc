:::info[適用對象]
收銀員及班次經理，用來執行每日店舖作業。
:::

## 開始前

- 收銀機已連線
- 員工帳號可用
- 印表機已準備好

## 作業檢查清單

<Tabs>
  <TabItem value="opening" label="開店">
1. 登入，並在需要時點選 `上班打卡`。
2. 開啟 `裝置` 並執行 `Test Print`。
3. 確認店名與營業日期／時間正確。
4. 開啟 `Ordering` 並確認桌台畫面已載入。
5. 開啟一次 `結帳`，確認付款方式可用。
  </TabItem>
  <TabItem value="during-shift" label="營運中">
1. 在確認品項後立即傳送訂單。
2. 收到印表機或網路警示後，重新檢查裝置狀態。
3. 只有在客戶要求時，才使用分單付款模式。
4. 在政策要求留痕時加入備註。
5. 指定一位負責人處理每一個付款事件。
  </TabItem>
  <TabItem value="handover" label="交接">
1. 在交接中的收銀機停止建立新訂單。
2. 完成或清楚交接進行中的結帳。
3. 確認待處理的作廢／退回項目已獲經理核准。
4. 把未解決的事件與交易編號一併交接。
  </TabItem>
  <TabItem value="closing" label="收店">
1. 處理所有未完成訂單與待處理付款。
2. 確認沒有收銀員卡在結帳處理中。
3. 完成 `代收 / 代支` 記錄。
4. 確認經理已準備好進行日結。
5. 執行日結並確認完成。
  </TabItem>
</Tabs>

## 良好的營運狀態

- 班次開始時沒有被阻擋的印表機或付款問題
- 每個事件都有明確負責人
- 日結時沒有未完成的結帳

## 如發生問題

:::warning[付款狀態不明確]
停止重複重試，並立即交由經理處理。
:::

:::warning[裝置仍然不可用]
先做一次基本檢查，然後直接交由經理處理，不要反覆嘗試。
:::

## 何時要找經理／管理員

- 必須變更 `收銀機代碼` 或其他關鍵設定
- 退貨／作廢需要核准
- 日結警告或驗證阻止關閉
